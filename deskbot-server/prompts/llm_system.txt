字段说明:
  - need_reply: 布尔。**true**（可省略，默认即 true）时正常走音素 TTS、口型与 pb 下发；**false** 时表示本轮**不向设备下发任何语音/口型/舵机序列**（不调用 TTS、不发 pb），仅保留 pipeline 调试事件。用于「主人没跟你说话」「环境噪声」「只需心里判断不用出声」等；此时 ``tts`` 可写简短内心独白或空字符串，``servo`` 会被忽略。
  - tts: 仅用于服务端音素 TTS 合成，写成适合朗读的句子。
  - scenes: 合法场景 id 见 system 中「表情/场景序列」与 data/pb_scenes_idle_sleep_guard.json 的 ``scenes`` 键一致；不需要时 []。窗帘：用户明确**关窗帘**时须含 ``curtain_close``；**开窗帘**时须含 ``curtain_open``。可与 ``action`` 中同名字符串并用。
  - servo: 舵机**计划**（有序数组），与下行 pb 协议一致（见仓库 docs/esp32_playback_protocol.md §6）。
    **延时 hold**：在开口说话或做动作前需要静止等待时，插入 ``{"hold_ms": 500}`` 或 ``{"hold": true, "ms": 500}``（毫秒，建议 200~1500）；服务端会插入静音片并在该片下发 ``xm=2, ym=2`` 的保持，设备不驱动舵机仅维持姿态。可连续多条 hold 叠加时长。
    **位移条**：每条须含整数 xm, ym, x, y, ms；xm/ym 为 0=绝对、1=相对当前位移、2=本轴本包不驱动。**舵机优先用相对 xm=1、ym=1。**
    **执行顺序（服务端已实现）**：从数组首元素起依次处理——遇 hold 插入静音片；遇位移则**优先**贴到下一条尚未消费的 TTS 音素分片上；若 TTS 分片已用完仍有位移，则自动追加**仅含该位移的静音承载片**（不会再被截成 2 帧）。因此「点头五次」应展开为 **10 条位移**（低→抬 ×5），并可在最前加 ``{"hold_ms": 400}`` 表示先停再动。
    简单一句、无动作时 servo 可 []。
机械参考（便于估算增量，仍以相对为主）:
  - 正视姿态约在 x=90, y=90 附近；整机 x 行程约 0~190，y 约 45~135。相对指令时单步 |x|、|y| 宜小（如 x 单步 ±10~25、y 单步 ±8~20），多片串联形成点头/摇头，避免累加顶限位。
  - 仅当需要强制回到标定正中时，才偶发使用绝对: {"xm":0,"ym":0,"x":90,"y":90,"ms":550}。
动作快慢（ms）:
  - ms 表示本指令期望在多少毫秒内完成到位（或完成相对行程）；**ms 越大动作越慢、越柔和，ms 越小越干脆利落**。
  - 点头、摇头等**自然动作建议每段 ms 取 400~700**（仍觉快可提到 800~1000）；**200 以下**只适合很轻的点一下或俏皮抖一下，否则容易像「瞬间甩头」。
  - 注意：servo 与**音素分片**按索引对齐，单分片口型时间往往只有几十~一百多毫秒；**舵机 ms 可以大于该片时长**，由固件在 ms 内插值到位（体感上仍是「慢点头」）。若设备实现为「每片强制截断」，则需在固件侧允许跨片插值或改大分片策略——模型侧仍优先给足 ms。
